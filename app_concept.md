---

## 1. App Overview

PreLoved Market is a mobile application designed to help students buy and sell second-hand clothing within their campus community.

The idea was developed based on identified challenges faced by students, including:
- Lack of a centralized platform for buying and selling clothes  
- Difficulty finding specific items  
- Uncertainty about sizes and fit  
- Low-quality product images  
- Lack of trust between buyers and sellers  

The app aims to address these challenges by creating a simple, trusted, and student-focused marketplace.

---

## 2. Target Users & Use Cases

### Primary Users
- University students (Year 1–4) who want to:
  - Sell clothes they no longer wear  
  - Buy affordable second-hand clothing  
  - Quickly find specific fashion items  

### Secondary Users
- Local thrift sellers near campus  
- Recent graduates  
- Campus staff  

### Key Use Cases

- **Selling Clothes:** Users upload images, add details (size, condition), and list items for sale  
- **Buying Clothes:** Users browse, search, and filter items to find what they need  
- **Finding Specific Items:** Users search directly (e.g., size 42 sneakers)  
- **Checking Size & Fit:** Users communicate with sellers to confirm measurements  
- **Sustainable Shopping:** Users buy second-hand items to save money and reduce waste  

---

## 3. Core Value Proposition

PreLoved Market provides a simple and trusted platform where students can easily buy and sell second-hand clothing within their campus.

### How the App Solves Problems

| Problem | Solution |
|--------|----------|
| No central platform | All listings are available in one app |
| Hard to find items | Search and filtering by size, price, and category |
| Size uncertainty | Size guide, measurements, and direct chat |
| Poor image quality | Requirement for multiple clear images |
| Lack of trust | Ratings and reviews system |

---

## 4. MVP Features (Minimum Viable Product)

### Feature Overview

| Feature | Description | Problem Solved |
|--------|------------|----------------|
| Product Listings | Users upload items with photos, size, price, and condition | Poor visibility |
| Search & Filter | Helps users find items easily | Difficulty finding items |
| Ratings & Reviews | Users rate sellers after transactions | Lack of trust |
| In-App Chat | Buyers and sellers communicate directly | Size and fit uncertainty |
| Size Guide | Helps users understand sizing | Wrong purchases |

### Why These Features Matter

- **Product Listings:** Ensures clear and complete item information  
- **Search & Filter:** Saves time and improves usability  
- **Ratings & Reviews:** Builds trust within the platform  
- **Chat:** Enables direct communication for clarification  
- **Size Guide:** Reduces returns and mismatched purchases  

---

## 5. Constraint Integration Plan

### Technical Constraints

| Challenge | Solution |
|----------|----------|
| Slow internet | Use lightweight images and optimized data loading |
| Device limitations | Support low-end smartphones |
| Limited data | Allow partial and offline-friendly viewing |

### Trust & Safety Constraints

| Challenge | Solution |
|----------|----------|
| Fake sellers | User ratings and reviews |
| Scams | Verified accounts using campus email |
| Fake images | Require multiple real product images |
| Unsafe communication | In-app chat system only |

### Time & Resource Constraints

| Challenge | Solution |
|----------|----------|
| Limited time | Focus on 5 core MVP features |
| No budget | Use cash-based transactions (no online payments) |
| Limited backend | Use SQLite database |
| Fast development | Use reusable Flutter components |

---

## 6. User Flow (Simple Overview)

1. User opens the app  
2. Browses or searches for items  
3. Views item details  
4. Chats with seller (if needed)  
5. Agrees on purchase  
6. Meets seller and completes transaction
